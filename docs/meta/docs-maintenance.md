# 文档贡献规范

目标：让文档可持续维护，避免“代码变了、文档没变”。

新增或调整功能时，请同步 **[开发手册](/guide/handbook)** 中对应章节（或增加新页并在 `.vitepress/config.mts` 的 `/guide/` 侧栏注册）。

## 基本原则

- 配置变更：同步更新 `configs`、`.env.example`、文档
- 命令变更：同步更新命令示例与参数说明
- 功能变更：至少补一段使用示例或行为说明；复杂流程用 **端到端步骤**（配置 → 代码路径 → 如何验证）
- 生产影响变更：同步更新运行手册与检查清单
- **表述范围**：以本仓库实现与直接依赖（Gin、GORM、Asynq、Viper 等）为准；**不写**与其它应用框架的对比、迁移对照或外链宣传，保持技术文档中性表述

## 推荐结构

- 指南类：放 `docs/guide/*`
- 运维类：放 `docs/ops/*`
- 路径类：放 `docs/paths/*`
- 规范类：放 `docs/meta/*`

## PR 自检（文档相关）

- [ ] 页面可在侧边栏找到
- [ ] 链接可点击且无 404
- [ ] 代码块命令可实际执行
- [ ] 文档构建通过：`npm run docs:build`

## 常用命令

```bash
npm run docs:dev
npm run docs:build
npm run docs:preview
```
