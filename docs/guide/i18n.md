# 本地化（i18n）

## 配置

`configs/*.yaml` 中 `i18n`：

- `default_lang`：默认语言标签，如 `zh`。
- `bundle_paths`：JSON 语言包路径列表，如 `./i18n/zh.json`、`./i18n/en.json`。

## 中间件

- `middleware.I18n` 在 `routes.Build` 中全局注册（见 [中间件参考](/guide/middleware-reference)）。
- 语言包为 **JSON**，结构需符合 `middleware/i18n.go` 中的解析约定（文件名中带语言段）。

## 在 Handler 中使用

- 业务响应优先通过 **`internal/pkg/i18n`** 的 `T(c, key, defaultMsg)` 与 `errcode` 的 key 搭配（见 `internal/api/response` 与 `error_helper`）。
- HTTP 客户端可传 **`Accept-Language`**（需在 `cors.allow_headers` 中放行）。

```go
import (
    "github.com/gin-gonic/gin"

    i18n "gin-scaffold/internal/pkg/i18n"
)

func exampleMsg(c *gin.Context) string {
    return i18n.T(c, "demo.greeting", "您好")
}
```

`messageID` 需在语言包 JSON 中存在对应键；否则返回 `defaultMsg`。

## 新增语言

1. 增加 `i18n/xx.json`。
2. 在配置 `bundle_paths` 中加入路径。
3. 重启服务或依赖热重载行为（配置热重载见 loader 说明）。

## 打包与部署（embed 行为）

- `i18n/*.json` 已通过 `go:embed` 内嵌到二进制。
- 运行时若磁盘没有 `i18n` 目录，会自动回退到内嵌资源（必要时释放到临时目录供中间件读取）。
- 结论：生产部署单二进制场景下，不强依赖手工拷贝语言包文件。

## 最小可复制示例

请求中文：

```bash
curl -H "Accept-Language: zh-CN" "http://localhost:8080/health"
```

请求英文：

```bash
curl -H "Accept-Language: en-US,en;q=0.9" "http://localhost:8080/health"
```

## 常见问题与排查

- 返回文案始终是默认语言：确认请求头 `Accept-Language` 已传入且 CORS 放行该 header。
- key 命中了默认消息：检查 `i18n/xx.json` 是否存在对应 key（大小写需一致）。
- 本地正常、线上异常：确认构建时 `i18n/*.json` 没被误删（否则 `go:embed` 编译期会失败）。

## 与错误码文档的关系

- 文案 key 与 [错误处理](/guide/error-handling) 中的约定应对齐，避免前端写死中文。
- 参数校验消息与 key 规则见 [验证器与多语言](/guide/validation-i18n)。
