# Security Policy

## 支持的版本

本仓库为**脚手架/模板**性质：请自行在衍生项目中评估升级策略。我们会尽量在 `main` / `master` 上通过 CI 保持依赖可构建，并对已确认的漏洞安排修复。

## 报告漏洞

如果你认为发现了**可利用的安全漏洞**（例如未授权访问、RCE、敏感数据泄露路径），请：

1. **不要**在公开 GitHub Issue 中发布细节或 PoC。
2. 通过仓库维护者认可的**私密渠道**联系（例如组织内部安全邮箱、或 GitHub Security Advisories，若仓库已开启）。

请尽量包含：

- 影响版本 / commit
- 复现步骤与影响范围
- 是否已有补丁思路（可选）

我们会在合理时间内确认并协调修复与披露节奏。

## 供应链与扫描

CI 包含 `govulncheck` 与 `gosec` 等静态检查；它们不能替代完整威胁建模。生产部署请结合网络隔离、密钥管理、最小权限与审计日志。
